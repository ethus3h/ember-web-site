Encoding

 Description:

This encoding is an attempt to address some of the issues in Unicode, and create an encoding to which adding characters is easier, using an extensible design, to enable adding of every character desired. The file ">Top.csv" contains a definition of the subencodings available. Two hundred eighty-nine character categories are currently supported. In the future, support is planned for non-unified Classical Yi, a native private-use area, and Arabic/Roman numerals in rectangles with specific corners rounded/missing. If you notice any errors or to request a character/script (anything goes!), please email me.
