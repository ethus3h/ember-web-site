How to fix strange permission issues when merging ebuilds — Wreathe Documentation

Page location: Software Wreathe Documentation Documentation for experimental features

This article applies to: Wreathe 7.3

The permissions of files set in Portage ebuilds sometimes don't get applied as expected when merging the package (installing the image into the system), with the consequent effect that fperms seems not to be working (seems not to be respected at merge time, or to be overridden or ignored, or to have no effect). This is probably happening because when files/directories are merged over top of existing files/directories, the permissions of the existing files/directories are used for the new ones, regardless of the permissions of the files being merged. The solution to this is to remove the files with the incorrect permissions before trying to merge the package. This is not something that needs to be fixed in the ebuild.
