This article applies to: Wreathe 7.2

Problem

Gnome-Do keyboard shortcut runs Gnome-Do twice

Can incorrectly appear to be related to this terminal output message:

(Do:11035): Wnck-CRITICAL **: wnck_set_client_type got called multiple times.


This message is apparently normal (see the bug report) and can "safely be ignored" (source) (See also this troubleshooting entry)

Solution

Have the keybinding execute the jack-do script instead; that script was included to resolve this issue.
