This page documents notes from the process of compiling the music Web site.

Miscellaneous observations

I don't think the thumbnail images or IA derivative files are really part of the releases, so they are not linked from the release page (except for MP3/OGA/FLAC transcodes). They all will be saved by crystallize-backup, though.

Not every session has a file representing its product, or a work version, since it might just be a commit in the git repository or something.

Empty config values I think will be treated as undefined, even if given in the file. An empty HTML comment as the value I think would produce an empty value if desired, although it might result in an empty link or other important navigational element. I've not implemented any solution for this yet, as it is only a hypothetical problem (I don't have any tracks that are supposed to have an empty name).

The artwork PNG files for Youtube releases are made from the thumbnail file saved by youtube-dl.

Only a few reference types (that look up information from a .conf file) are implemented (context, work, artist, etc.).

For the "label" attribute of artists, I've supplied the most recent label. Look at releases for more comprehensive label information.

Field meanings

fileIsSemanticallySignificant
Documents whether the file format of the music should be considered part of the artistic work. Generally if it is not given, that means "no".
youtubeSourceFile
The name of the original file that was uploaded to YouTube.
ytdled
ZIP file containing a YouTube video downloaded using youtube-dl.
ytdledFlac
A YouTube video as downloaded using youtube-dl and transcoded to FLAC.
