BeHeMMOth

Page location: Home Software Wreathe Subprojects

Overview

A bullet-hell computer game with support for multiple players over the Web. It is a fork of Fabiano, and consequently is licensed under the AGPL3.

The client is not yet confirmed to work without non-libre dependencies (it so far has only been successfully tested with Adobe Flash and the freshplayerplugin wrapper). Resolving this issue is a high priority. In addition, some of the "source code" for the project is actually binary blobs, due to the inclusion of precompiled dependencies and such; eliminating those is another high priority.

Installation

For Wreathe 7.3 and similar operating systems (e.g. Gentoo GNU/Linux), an ebuild (games-roguelike/behemmoth) is available in the Wreathe overlay.

For other operating systems, use the following installation instructions.

Download

To download BeHeMMOth, run:

git clone https://github.com/ethus3h/BeHeMMOth.git

Requirements

  • See the ebuild for details.

Compilation

cd into the source code directory, and run make.

Installation

Run sudo make install.

Troubleshooting

If the client gives a "Error loading language" error, it was probably unable to connect to the server. Make sure the server is accessible to the client. If the client just gives a black screen, the language file (e.g. en.txt) probably has an incorrect syntax (it must be a UTF-8 file in the form [["key1", "value", "languageAbbreviation"], ["key2", "value", "languageAbbreviation"]] with a byte order mark at the beginning; incorrect whitespace (e.g. newlines) or use of commas will probably cause problems). If the client says "Loading..." on a grey background on a black screen, make sure it has been added to the Adobe Flash trust list ("Global Settings" in the context menu). If when you try to enter a realm or the Nexus it takes a long time, just give it a minute; it may say "Socket-Server Security Error: Error #2048" and "Connection failed! Retrying...": this is a known issue but doesn't seem to adversely affect use of the software.
